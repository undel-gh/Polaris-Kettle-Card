# Kettle Card

Единая Lovelace-карточка для умного чайника в Home Assistant: запуск/остановка нагрева,
выбор режима нагрева, выбор напитка (пресета), управление целевой температурой,
переключатели (питание, подсветка, звук, блокировка, ночник) и статус устройства
(доступность, RSSI, прошивка, ошибки) — всё на одной карточке.

Карточка не требует сборки: чистый Custom Element без внешних зависимостей
(кроме встроенных `ha-icon` / `ha-entity-picker` из самого Home Assistant).

## Установка

### Вручную
1. Скопируйте `kettle-card.js` в `config/www/kettle-card.js`.
2. Настройки → Панели → Ресурсы → Добавить ресурс:
   - URL: `/local/kettle-card.js`
   - Тип ресурса: JavaScript-модуль

### Через HACS (кастомный репозиторий, до публикации в общем каталоге)
1. HACS → Frontend → меню (⋮) → Custom repositories.
2. Добавьте URL репозитория, категория **Lovelace**.
3. Установите `Kettle Card`, ресурс подключится автоматически.

## Конфигурация

Минимально нужна только сущность `water_heater`. Остальные — опциональны:
если не указать, соответствующий блок карточки просто не отобразится.

```yaml
type: custom:kettle-card
name: Чайник
entities:
  water_heater: water_heater.kettle_pwk_1725cgld_water_heater
  power: switch.kettle_pwk_1725cgld_power
  temperature: sensor.kettle_pwk_1725cgld_temperature
  mode_select: select.kettle_pwk_1725cgld_select_mode_kettle
  night_light: light.kettle_pwk_1725cgld_night
  backlight: switch.kettle_pwk_1725cgld_backlight
  child_lock: switch.kettle_pwk_1725cgld_child_lock
  sound: switch.kettle_pwk_1725cgld_sound
  available: binary_sensor.kettle_pwk_1725cgld_available
  error: sensor.kettle_pwk_1725cgld_error
  rssi: sensor.kettle_pwk_1725cgld_rssi
  firmware: sensor.kettle_pwk_1725cgld_firmware_version
  device_type: sensor.kettle_pwk_1725cgld_device_type
```

Через UI-редактор (значок карандаша при добавлении карточки) все сущности можно
выбрать через стандартные пикеры Home Assistant — редактировать YAML вручную не обязательно.

### Дополнительные параметры

| Ключ             | Назначение                                                                 |
|------------------|-----------------------------------------------------------------------------|
| `default_mode`   | Режим, который включается кнопкой «Вскипятить» (по умолчанию `electric`).  |
| `preset_labels`  | Переопределение подписей/иконок напитков, ключ — значение `select.options`.|
| `mode_labels`    | Переопределение подписей/иконок режимов нагрева (`operation_list`).        |

```yaml
mode_labels:
  performance:
    label: Быстро вскипятить
    icon: mdi:fire
preset_labels:
  black_tea:
    label: Чёрный чай
    icon: mdi:tea
```

> Названия режимов нагрева (`off / performance / electric / heat_pump / eco`)
> соответствуют пунктам на самом чайнике: Выключен, Кипячение, IQ Кипячение,
> Разогрев с удержанием, Разогрев. При необходимости их можно переопределить
> через `mode_labels`.

## Что показывает карточка

- **Сосуд с «водой»**: заливка меняется от синего (холодная) к оранжевому
  (кипяток) по показанию датчика температуры; во время нагрева поднимается
  анимированный пар.
- **Крупный цифровой индикатор** текущей температуры (моноширинный шрифт —
  как на экране самого чайника) и целевой температуры.
- **Кнопка «Вскипятить/Остановить»** — управляет `water_heater` напрямую.
- **Шаг температуры** (±) с учётом `min_temp`/`max_temp`/`target_temp_step`.
- **Чипы режимов нагрева** и **чипы напитков** (из `select.options`).
- **Переключатели**: питание, подсветка, звук, блокировка, ночник.
- **Индикатор связи** (точка + иконка Wi-Fi с уровнем по RSSI) и баннер
  ошибки, если `sensor.error` ≠ `no_error`.
- Клик по названию, температуре, статусу или значениям в подвале (прошивка/
  RSSI/тип устройства) открывает стандартный диалог "Подробнее" (`more-info`)
  соответствующей сущности.

## Совместимость с темами

Все цвета, кроме акцентного градиента «вода», берутся из переменных темы
Home Assistant (`--card-background-color`, `--primary-text-color`,
`--divider-color` и т.д.), поэтому карточка одинаково хорошо смотрится
в светлой и тёмной теме.

## Публикация в HACS

Для публикации в основном каталоге HACS понадобится:
1. Отдельный публичный GitHub-репозиторий с этим кодом.
2. Файл `hacs.json` (см. в этой папке) с корректным `name`.
3. `README.md` (этот файл) и лицензия.
4. Релиз (git tag) — HACS ставит карточки по релизам, а не по `main`.

## Ограничения / TODO

- Ночник (`light`) переключается только вкл/выкл; управление яркостью и
  цветом делается через диалог "Подробнее" (клик по названию не выведен для
  ночника отдельно — при желании легко добавить попап с ползунком).
- Расшифровка кодов `sensor.error` не встроена (кроме `no_error`) — при
  необходимости пришлите список возможных значений, и мы добавим карту
  переводов.
